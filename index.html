Gestionerete

Il progetto "Gestionerete" è un sistema di gestione e monitoraggio della rete aziendale progettato per semplificare il controllo dei dispositivi di rete, degli utenti e degli allarmi.

Funzionalità principali

  • Monitorare lo stato dei dispositivi (Online/Offline)
  • Gestire gli allarmi generati da dispositivi non funzionanti
  • Associare utenti ai dispositivi per una migliore tracciabilità
  • Automatizzare il rilevamento di problemi tramite eventi schedulati

Tecnologie utilizzate

  • MySQL per il database
  • PHP per il backend
  • Eventi e procedure SQL per l'automazione
  • Packet Tracer per la simulazione della rete

Funzionalità Avanzate

Monitoraggio Automatico

  • PuliziaGiornalieraAllarmi: Ogni giorno ripulisce la tabella allarmi da allarmi risolti

    • Evento Schedulato - PuliziaGiornalieraAllarmi

    Ogni giorno ripulisce la tabella allarmi da allarmi risolti:

    UPDATE Dispositivo
  SET Stato = CASE Stato
                WHEN 'Online' THEN 'Offline'
                WHEN 'Offline' THEN 'Online'
              END
  WHERE Tipo_Dispositivo = 'PC'
    AND RAND() < 0.5
  • Procedura (ControllaPCOffline): Genera automaticamente allarmi per i PC offline

    • Routine - ControllaPCOffline

    Genera automaticamente allarmi per i PC offline:

    BEGIN
    -- Dichiarazione variabili
    DECLARE done INT DEFAULT 0;
    DECLARE pc_id INT;
    DECLARE pc_stato VARCHAR(10);
    DECLARE cur CURSOR FOR SELECT ID_D, Stato FROM Dispositivi WHERE Tipo_Dispositivo = 'PC';
    DECLARE CONTINUE HANDLER FOR NOT FOUND SET done = 1;
    
    -- Apertura cursore
    OPEN cur;
    
    -- Ciclo attraverso tutti i PC
    pc_loop: LOOP
        FETCH cur INTO pc_id, pc_stato;
        IF done THEN
            LEAVE pc_loop;
        END IF;
        
        -- Cambia stato casualmente (50% di probabilità)
        IF RAND() < 0.5 THEN
            IF pc_stato = 'Online' THEN
                UPDATE Dispositivi SET Stato = 'Offline' WHERE ID_D = pc_id;
                
                -- Registra allarme se il PC va offline
                INSERT INTO Allarmi (Tipo, Risolto, Data_Allarme, Data_Ris, ID_D)
                VALUES ('PC Offline', FALSE, NOW(), NULL, pc_id);
            ELSE
                UPDATE Dispositivi SET Stato = 'Online' WHERE ID_D = pc_id;
                
                -- Aggiorna allarme se esiste e non è risolto
                UPDATE Allarmi 
                SET Risolto = TRUE, Data_Ris = NOW() 
                WHERE ID_D = pc_id AND Tipo = 'PC Offline' AND Risolto = FALSE;
            END IF;
        END IF;
    END LOOP;
    
    -- Chiusura cursore
    CLOSE cur;
END

Descrizione della Rete Simulata

Dispositivi principali

  • Smartphone (SMARTPHONE-PT): Dispositivi mobili collegati alla rete
  • PC: Workstation collegati agli switch
Struttura:
  • Ogni smartphone è associato a un access point dedicato (es. Smartphone1 ↔ Access Point-progettazione)
  • Ogni switch gestisce 3 PC fissi, suggerendo una suddivisione logica per reparto

Integrazione con il Sistema "gestionerete"

La rete simulata può essere monitorata tramite il database gestionerete:

  1. Mappatura dei dispositivi:
    • Gli Smartphone e i PC possono essere registrati nella tabella Dispositivo con:
      • Tipo_Dispositivo: Smartphone o PC
      • Stato: Impostato a Online/Offline in base alla connettività
Monitoraggio:
  • L'evento CambiaStatoPC può simulare guasti casuali nei PC collegati
  • La procedura ControllaPCOffline genererà allarmi se un PC o un access point risulta offline

Vantaggi del Sistema

  • Monitoraggio centralizzato della rete aziendale
  • Allarmi automatici per dispositivi non funzionanti
  • Scalabile con possibilità di aggiungere nuovi dispositivi e funzionalità

GuardiaPerimetrale

Il progetto "GuardiaPerimetrale" è un sistema web-based avanzato per la gestione e il monitoraggio dei dispositivi di rete all'interno di un'infrastruttura IT.

Funzionalità principali

  • Aggiungere, modificare ed eliminare dispositivi (firewall, server, router, workstation, switch)
  • Monitorare lo stato (online/offline/warning) con aggiornamenti in tempo reale
  • Visualizzare e gestire gli allarmi relativi alla sicurezza e alle prestazioni
  • Generare report dettagliati sullo stato della rete
  • Interfaccia intuitiva con dashboard personalizzabile

Tecnologie utilizzate

  • PHP per la logica di backend con architettura MVC
  • MySQL per il database con ottimizzazioni delle query
  • HTML5/CSS3 per un'interfaccia utente responsive
  • JavaScript per interattività e aggiornamenti in tempo reale

Architettura del Sistema

Il sistema è stato progettato con un'architettura modulare che ne facilita la manutenzione e l'espansione:

Frontend

  • Interfaccia responsive basata su HTML5/CSS3 con layout a griglia
  • Navigazione intuitiva con menu a schede e breadcrumb
  • Feedback visivi immediati (badge colorati, notifiche toast)
  • Grafici interattivi per la visualizzazione dei dati

Backend

  • PHP con pattern MVC per una chiara separazione delle responsabilità
  • API RESTful per l'integrazione con altri sistemi
  • Middleware per l'autenticazione e l'autorizzazione
  • Gestione degli errori centralizzata

Sicurezza

  • Protezione contro SQL Injection tramite prepared statements
  • Validazione lato server di tutti gli input
  • Hashing delle password con algoritmi moderni (bcrypt)
  • CSRF protection e sanitizzazione dell'output
  • Controllo degli accessi basato su ruoli (RBAC)

Vantaggi del Sistema

  • Centralizzazione: Tutte le informazioni di rete in un'unica piattaforma
  • Monitoraggio in tempo reale: Notifiche immediate per problemi critici
  • Scalabilità: Architettura modulare che supporta l'aggiunta di nuove funzionalità
  • Accessibilità: Interfaccia intuitiva per utenti con diversi livelli di competenza
  • Reporting avanzato: Generazione automatica di report periodici
